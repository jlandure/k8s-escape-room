# Kubernetes Escape Room Game

You are a wizard apprentice. While your teacher is out of house you decided to sneak into his study to practice some spells. 
Maybe try out some magic artifacts you find there. 
But while reciting an incantation you found in a hidden tome something went horribly wrong. 
Important things are missing or not working anymore. It is now on you to fix what has been broken.

**Broken parts in the study are marked with a red circle. 
Hovering there with your cursor gives you a hint for what you might be looking for.**

## Instructions etudiants (GitHub Codespaces)

Chaque etudiant dispose de son propre environnement isole, avec son propre cluster Kubernetes jetable. Aucune installation locale n'est requise.

1. **Ouvrir le Codespace** via ce lien (une machine 2-core suffit) :
   [Creer mon Codespace](https://github.com/codespaces/new?template_repository=jlandure/k8s-escape-room)
   Le reseau `kind` et le forwarding des ports 80/443 sont configures automatiquement.

2. **Demarrer le jeu** : dans le terminal du Codespace, lancer les deux commandes suivantes et patienter quelques minutes le temps de l'installation :
   ```bash
   ./workbench.codespaces
   . init.sh
   ```

3. **Ouvrir la piece** : aller dans l'onglet **Ports**, puis ouvrir le port **80** (ou 443) dans le navigateur. La salle s'affiche avec les zones cassees entourees de rouge.

4. **Jouer** : reparer les ressources cassees avec [k9s](https://k9scli.io/) ou [kubectl](https://kubernetes.io/docs/reference/kubectl/). Survoler les ronds rouges affiche un indice. Le frontend reflete la progression.

5. **A la fin** : arreter ou supprimer le Codespace depuis [github.com/codespaces](https://github.com/codespaces) pour stopper la facturation.

> Astuce : sous Firefox, les info-bulles n'apparaissent que si la fenetre du navigateur est au premier plan. Prefere un autre navigateur.

## Get started
Regardless of execution environment, there are certain tools you can use to interact with the escape room.
Once you are in the workbench and have initialized the cluster with `. init.sh`, you can use [k9s](https://k9scli.io/) to see cluster resources in the terminal.
You can also use [kubectl](https://kubernetes.io/docs/reference/kubectl/).

### Note
Tooltip hints are available when hovering with the cursor over the red circles. Note that in the firefox browser the tooltip hints only appear when the browser window is focused.
Please use another browser or keep in mind to focus the browser window for tooltips when using firefox.
